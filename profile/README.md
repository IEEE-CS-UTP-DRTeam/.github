# IEEE Computer Society UTP — Development & Applied Research Team (DRTeam)

Somos el equipo técnico del capítulo estudiantil **IEEE Computer Society UTP**. Construimos software e investigación aplicada con entregables reales para aumentar el **impacto** y la **visibilidad** del capítulo, mientras formamos miembros con prácticas profesionales de ingeniería.

---

## Qué hacemos

- **Herramientas para el capítulo (Chapter Tech)**
- **Proyectos académicos e institucionales**
- **Proyectos experimentales** (IA, videojuegos, Open Source)

---

## Resultados esperados

- Mayor impacto técnico y visibilidad del capítulo estudiantil
- Miembros con formación en:
  - Habilidades técnicas y blandas
  - Git, control de versiones y colaboración en equipo
  - Buenas prácticas de ingeniería de software
- Proyectos con entregables claros (demo, release, documentación) en las 3 líneas del equipo

---

## Cómo contribuir / Participar

### Requisito para pertenecer
Para pertenecer al DRTeam debes estar **involucrado y participando activamente** dentro de **IEEE Computer Society UTP**.

### Flujo de trabajo (rápido)
- Trabajo = **Issues**
- Cambios = **PR**
- Merge con al menos **1 review**
- **CI** cuando aplique (lint/tests/build)

---

## Recursos útiles

- Organización en GitHub: https://github.com/IEEE-CS-UTP-DRTeam
- Documentación y guías: revisa el repositorio `handbook` o la carpeta `docs/` de cada proyecto

## Estándares mínimos por proyecto

- `README.md` con instalación/uso y (si aplica) demo
- `CONTRIBUTING.md` para contribuciones
- Estructura mantenible y PRs pequeños
- Documentación mínima (`docs/`) cuando aplique
- Licencia (`LICENSE`) si el repositorio es público / open source
